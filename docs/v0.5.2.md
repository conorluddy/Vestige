# V0.5.2 — Menu-bar controls

V0.5.1 shipped a **read-only** macOS menu-bar app over the daemon's status file. V0.5.2 makes
it a **control surface**: it can pause, kick, reload, and toggle the daemon, opens a persistent
project window, and is easy to boot — without ever touching `launchctl`. It is the second
phase of issue #88, plus the folded-in **EXP-1..7** experience layer (#114–#120).

The split that matters for this repo: the **Rust half** (IPC + CLI) is fully tested and runs
on every platform; the **Swift half** (the macOS app) is authored against the existing
`app/Vestige-Mac/` project and needs a macOS build (`app/Vestige-Mac/scripts/build-app.sh`)
plus the manual test matrix — it cannot be compiled in CI.

## Pause is tick suppression (Rust)

`daemon.pause` sets an in-memory `paused_until` and pushes it to the scheduler over a `watch`
channel. While `now < paused_until` the embed / prune / TTL / session-scan ticks are skipped;
the 5-second status tick **always** runs and writes `paused_until` so observers see the pause.
Pause auto-expires — a past instant reads as not-paused, so ticks resume with no explicit
resume. In-flight jobs complete (tick-boundary semantics). Pause is in-memory: a launchd
restart clears it. `schema_version` stays `1` (the `paused_until` field is additive).

```bash
vestige daemon pause --for 1h          # or --until 2026-06-04T08:00:00Z
vestige daemon status                  # shows paused_until
vestige daemon resume                  # clears it (idempotent)
vestige daemon kick scan               # V0.5.4 session scan, on demand
```

`--for` accepts compact durations (`30s`, `15m`, `1h`, `2d`); the CLI converts it to an
absolute UTC instant before sending, so the daemon only ever sees an absolute `until`.

## Booting the app (Rust)

```bash
vestige ui            # launch Vestige.app (macOS only)
vestige ui --login    # launch and register the SMAppService Login Item
```

After a successful `vestige init` or `vestige daemon install`, Vestige offers — **once, only in
an interactive macOS terminal** — to launch the app and start it at login. The prompt is
strictly gated by `commands::ui::decide_boot` (the agent-safety invariant): it never fires when
not macOS, in CI, when `VESTIGE_NONINTERACTIVE` is set, on a non-TTY, or with `--no-ui`. `--yes`
accepts it non-interactively for scripted *human* setup. A coding agent running `vestige init`
never spawns a GUI.

## The menu-bar control surface (Swift)

- **Socket client** (`DaemonSocket.swift`) — newline-delimited JSON-RPC 2.0 over
  `~/.vestige/daemon.sock` for kick / pause / resume / reload.
- **Actions** (`MenuView` + `DaemonActions`) — kick embed, scan sessions, pause (1h / until
  morning), resume, reload config; `vestige` shell-outs for the daemon enable/disable toggle,
  open browser/log/doctor; a `Start at login` toggle backed by `SMAppService`.
- **Pause affordance** — the menu-bar glyph switches to a paused icon and the header shows
  `Paused · resumes in …`.
- **Persistent workspace window** — a real `Window` (not just the popover) listing projects.

## Experience layer (EXP-1..7, Swift)

| # | Feature | Notes |
|---|---------|-------|
| EXP-1 | Activity pulse | `StatusFileWatcher.activityDetected` diff → `symbolEffect` bounce on real work. Unit-tested. |
| EXP-2 | Inbox badge + review | candidate-count badge on the icon + a row opening `vestige browse --tab candidates`. |
| EXP-3 | Polish | `ProjectHealth` centralises status-dot semantics; `.regularMaterial` backdrops (Liquid Glass is the macOS-26 follow-up); a11y identifiers. |
| EXP-4 | Sparkline | app-local memory-count history (UserDefaults ring buffer) → `Sparkline` in the expanded row. |
| EXP-5/6 | Memory peek | recent-memories ticker + ⌘-search via a **CLI shell-out** bridge (`MemoryBridge`), scoped to the active project — no new socket surface. |
| EXP-7 | Quick capture | ⌥⌘N global hotkey → capture window → `vestige note add`. The one new GUI write path, through core APIs only. |

**Resolved open questions (#88, pragmatic defaults):** read/write bridge = CLI shell-out;
active project = the supervised project with the most recent `last_memory_at`; sparkline history
= app-local buffer (no daemon schema change). The global-hotkey Carbon hardening and
active-project edge cases (front-app cwd vs last-focused) remain follow-ups.

## Invariants

No new memory write path on the daemon (pause only suppresses scheduling); EXP-7's capture and
all reads route through existing `vestige` core APIs, never raw SQL. Soft-delete, project-scope,
immutable-migration, and additive-schema rules all hold.
